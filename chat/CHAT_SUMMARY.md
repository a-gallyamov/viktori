## 2. Плата 2.3.1 — отличия от старых версий

### Эволюция плат

| Версия | Описание |
|--------|----------|
| 1.6.1 | Текущая стабильная (описана в PDF) |
| 1.6.2 | Улучшенная разводка, земляной полигон |
| 1.6.2mini | Компактная версия (в разработке) |
| 1.6.3 | Версия Shamane |
| 2.3 | P-канальный MOSFET, биполярное управление |
| **2.3.1** | **Последняя**. N-канальный Q1, HCPL3120 драйвер, INA226 Alert |

### Ключевые изменения в 2.3.1

- **N-канальный Q1** вместо P-канального IRF4905 → меньше потерь, выше КПД
- **HCPL3120** — оптоизолированный драйвер затвора (требует изолированного питания, B0515S)
- **INA226 Alert** — аппаратная токовая защита через Alert-линию → управляет Q3/Q4
- **Новые назначения пинов**:
  - A0 — защита входного питания (вместо A2)
  - A2, A3 — NTC термисторы
  - A6 — конфигурация отличается от 2.3
  - A7 — делитель напряжения БП (**6.2кОм + 1кОм** вместо 10кОм + 1кОм)
- SMD-компоненты формата 0805/1206
- Изолированные площадки под Q1 и VD2 (не соединены с землёй, несмотря на вид)

### Особенность HCPL3120

- **UVLO (Undervoltage Lockout)**: если питание < 12.3В при старте, нижний FET открывается → напряжение заряда появляется на выходе
- Решение: отдельный изолированный БП для N-канальной версии

## 3. Новые функции (после PDF v1.4)

### Режим БП (PSU)

- Работа как лабораторный регулируемый источник питания
- Стабильное поддержание напряжения без подключённого АКБ
- Блокировка модуля защиты в режиме БП (v1.43.2+)
- Два параметра контроля напряжения: предел напряжения заряда + мониторинг входного напряжения

### Логгер

- **ESP8266/WeMos D1 Mini** + Data Logger Shield (RTC + MicroSD)
- Веб-интерфейс по адресу 192.168.1.116
- Запись CSV: интервалы 1, 10 или 20 секунд
- **Важно**: подключать только TX от зарядки (RX вызывает перезагрузку на v1.38+)
- Просмотр графиков: Python (Grafik_Monitor_v8.py) или .exe программа
- Автомасштабирование графиков

### ESP32 (экспериментально)

- ESP32-S3 Nano — предпочтительный вариант
- WiFi/BLE для удалённого мониторинга
- **Ограничение**: WiFi занимает ADC2 → безопасные пины только A0-A3 (ADC1)
- Требуется переназначение пинов D2/D3 (D3 должен быть HIGH при загрузке ESP32)

### Улучшенные алгоритмы

- **Дозаряд по dV/dt** — порог 500 мВ (define Volt_difference_mV), таймер 30 мин (define Time_AddCharge 1800)
- **Буферный/хранение** — стабильное поддержание напряжения (~100 мА)
- **Мягкий старт** для токов < 800 мА
- **Минимальный порог тока** < 10 мА для отключения заряда
- **CORRECTCUR** — коррекция тока при росте напряжения (включён по умолчанию)
- **Подавление выброса напряжения** при отключении АКБ во время заряда (v1.40+)

### Прочее

- **MicroUART** — замена Serial, экономия 500 байт flash + 200 байт RAM
- **INA226 BIT_SOL** — аппаратная защита от превышения тока (v1.54+)
- **Режим паузы при отключении питания** вместо ошибки
- **Поддержка LGT8F328P** — бюджетная альтернатива Arduino с большим объёмом памяти
- **10 профилей** АКБ с независимыми настройками
- **Поддержка дисплеев 20×4** с кириллицей

## 4. Частые проблемы и решения

### INA226

| Проблема | Решение |
|----------|---------|
| «INA226 error» при старте | Бракованный модуль с AliExpress — заменить |
| Перепутаны SDA/SCL на модуле | Проверить порядок пинов от GND: если GND→SDA→SCL — джамперы 1-2, 4-5; если GND→SCL→SDA — джамперы 2-3, 5-6 |
| Дрейф показаний | Дроссель в линии питания INA226; конденсатор 100-1000мкФ на 5В |
| Неверный адрес | I2C сканер скетч; типичные адреса: 0x40, 0x44, 0x45 |
| Шум от коммутации | Отдельное питание для INA226; I2C провода подальше от силовых цепей |

### Контрафактные компоненты

| Компонент | Как распознать |
|-----------|---------------|
| IRF4905 | Покупать только на ChipDip.ru; на AliExpress массово подделки |
| INA226 | Некоторые партии нерабочие — проверять I2C сканером |
| Arduino Nano | Нет кварца рядом с чипом = LGT8F328 вместо ATmega328P |
| Электролитические конденсаторы | Китайские дешёвые выходят из строя в импульсных цепях |

### Калибровка

| Проблема | Решение |
|----------|---------|
| Калибровка слетает при обновлении | Перекалибровка после каждого обновления прошивки |
| v1.39.4: калибровка БП не работает | Добавлен параметр VREF обратно в следующих версиях |
| Ток заряда занижен на 60-80 мА | Проблема мультиметров UNI-T UT71A на малых токах; INA226 точнее |
| Разные значения делителя | Измерить реальные сопротивления R10/R6, обновить #define в скетче |

### Калибровка — процедура

1. **Напряжение**: измерить вход мультиметром → установить такое же в калибровке → коэффициент INAVOLTKOOF (default 250)
2. **Ток**: подключить нагрузку (лампа 55Вт) + точный амперметр → подогнать шунт
3. **Порядок**: сначала ток, потом напряжение
4. **Нагрузка**: 21Вт недостаточно — использовать 55Вт
5. **Сброс калибровки**: удержание энкодера при входе в калибровку

### Дроссель

| Проблема | Решение |
|----------|---------|
| Перегрев | Жёлто-белые кольца (mix -26) хуже; использовать мятные/голубые (mix -52) |
| Насыщение при >6A | Нужен L=20-60 мкГн; 26 витков 1.2мм провода на кольце 27×12мм ≈ 41-47 мкГн |
| Чёрные кольца | Проницаемость ~1500 — быстро насыщаются, не использовать |
| Источник хороших сердечников | Извлекать из старых ATX БП 300-350Вт |

### Прочие проблемы

| Проблема | Решение |
|----------|---------|
| Заряд не запускается (ток = 0) | Проверить сигнал на D3; проверить оптопару |
| Ложная ошибка «Q1 пробит» | Реализовать двойную проверку (2 подряд срабатывания) |
| Экран зависает | Конденсатор 0.1 мкФ между +5В и GND на LCD; I2C провода подальше от силовых |
| Квадраты на дисплее | Нужен OLD bootloader в настройках Arduino IDE |
| Логгер вызывает перезагрузку | Убрать линию RX, использовать только TX |
| Минимальный ток 150 мА для старта | Настроить CURMIN в конфигурации |

## 5. Инциденты безопасности

### Критические

1. **Arduino сгорел при прошивке с подключённым БП к 220В**
   - Стабилизатор 5В вышел из строя → 12В пошло на USB → сгорел USB-порт компьютера
   - **Всегда отключать БП от сети при прошивке через USB**

2. **Дым от INA226 при неправильном шунте**
   - Шунт 0.089Ω вместо 0.0089Ω → повреждение чипа
   - **Всегда проверять сопротивление шунта перед установкой**

3. **Выход из строя AMS1117 от перегрузки**
   - Дешёвые AMS1117 не выдерживают нагрузку (вентилятор + дисплей)
   - **Использовать отдельные DC-DC для Arduino (5В) и вентилятора (12В)**

4. **Скачки напряжения до 800В**
   - При обратном смещении импульсного БП
   - **Трансформаторный БП надёжнее; добавить TVS-диод P6SMB6.8A**

### Рекомендации

- Предохранитель на входе БП (сторона 220В)
- Защита от переполюсовки на клеммах АКБ (разъём Anderson или ключевой)
- Радиаторы на все силовые компоненты + активное охлаждение
- Не работать с открытым 220В — корпус обязателен
- USB изоляция при прошивке под напряжением (гальванический изолятор)

## 7. Советы по сборке

### Дроссель

- **Рекомендуемые сердечники**: мятно-зелёные/голубые кольца (mix -52) — лучше на высоких частотах
- **Жёлто-белые** (mix -26): допустимы, но потери выше
- **Чёрные**: не использовать (проницаемость ~1500, быстро насыщаются)
- **Параметры**: 20-60 мкГн при 10А; 26 витков 1.2мм провода на кольце 27×12мм
- **Источник**: извлекать из старых ATX БП 300-350Вт
- **Расчёт**: radioskot.ru, coil32.net, gengen.ru

### Пайка и монтаж

- I2C провода — **короткие** (<20 см) и подальше от силовых цепей
- Проверить порядок GND/SDA/SCL на INA226 **до** пайки
- Конденсаторы на 12В и 5В рельсах — несколько штук
- Дорожки затвора — короткие и прямые к MOSFET
- При токе >6А: утолщить дорожки проводом 1-1.5мм

### Первое включение

1. Без INA226: проверить меню, энкодер, кнопки
2. С INA226: должно появиться напряжение и ток; если нет — I2C сканер
3. **Не замыкать** выход для тестирования КЗ-защиты при первом включении — использовать токоограниченный лабораторный БП

### Корпус

- Корпуса от ИБП — подходят отлично (можно использовать разъёмы)
- Пластиковые коробки Kradex
- 3D-печать (PLA/PETG) — STL-файлы в сообществе
- Радиаторы: 30-40мм алюминиевые (10×15×30мм)
- Вентилятор: 3-пиновые от ПК БП (нужны только 2 провода: GND + 12В)

### Охлаждение

- До 5А: достаточно радиаторов без вентилятора
- Выше 6А: обязательно активное охлаждение
- Термопаста между транзистором и радиатором
- Датчик температуры крепить к Q1 или дросселю (дроссель греется сильнее)

## 8. Конфигурация прошивки (ключевые define)

```c
#define LANG 0              // 0=русский, 1=English, 2=транслитерация
#define DISPLAY 0           // 0=16x2, 1=20x4
#define VOLTIN 1            // 0=без делителя, 1=с делителем
#define INA226_INCLUDED 1   // Включить INA226
#define TEMP_INCLUDED 1     // Включить датчик температуры
#define FAN 1               // 0=выкл, 1=вкл, 2=ШИМ
#define DEG_ON 40           // Температура включения вентилятора (°C)
#define DEG_OFF 37          // Температура выключения (°C, гистерезис)
#define PWM_FREQ 31         // 31 или 61 кГц
#define SHUNT 0.008f        // Сопротивление шунта (Ом)
#define CURR_MAX 10f        // Максимальный ток (А)
#define VOLT_MAX 26         // Максимальное входное напряжение (В)
#define DIV_R1 18000        // Верхний резистор делителя (Ом)
#define DIV_R2 1000         // Нижний резистор делителя (Ом)
#define INAVOLTKOOF 250     // Коэффициент INA226

// Новые define (v1.38+)
#define CORRECTCUR           // Коррекция тока при росте напряжения
#define INA226_BIT_SOL       // Аппаратная защита INA226 (v1.54+)
#define Volt_difference_mV 500  // Порог dV для дозаряда
#define Time_AddCharge 1800     // Таймер дозаряда (секунды)
#define VKORRECT             // Обязателен для корректной компиляции
```

### Для платы 2.3.1

```c
#define DIV_R1 6200         // Делитель 6.2кОм (вместо 10кОм/18кОм)
#define DIV_R2 1000         // Нижний резистор 1кОм
#define PROTECT A0          // Пин защиты (вместо A2)
```

## 11. Результаты тестирования АКБ из чата

### Десульфатация

- Сильно разряженный 60Ач Ca/Ca восстановлен до 49.5 А/ч за 24ч, полная ёмкость за 48ч+
- АКБ с КЗ банкой (108мОм внутреннее) удалось восстановить до рабочего состояния
- Ключ: терпеливый многоцикловый подход, не агрессивный перезаряд

### Рекомендации по зарядке

| Тип АКБ | Напряжение | Буферное | Примечание |
|---------|-----------|----------|------------|
| Pb Ca/Ca | 14.7В | 13.5-13.8В | Стандартная конфигурация |
| AGM | 14.7В | 13.5-13.8В | Чувствителен к температуре |
| LiFePO4 | 14.4В (3.6В/яч.) | — | Не допускать trickle-заряд |

### Измерение внутреннего сопротивления

- Тест лампой дальнего света от ВАЗ
- Новый АКБ 60Ач: ~4мОм → ~725А пусковой (при заявленных 820А — норма)
- Точность зависит от нагрузки и температуры

## 12. Частоты ШИМ — практика

| Частота | Плюсы | Минусы |
|---------|-------|--------|
| 4 кГц | Простота | Слышимый свист |
| 8 кГц | Лучший КПД | Тише, но всё ещё слышен |
| 31 кГц | Рекомендуется | Минимальный шум |
| 61 кГц | Меньший дроссель | Выше коммутационные потери |

## 13. Блок питания — рекомендации из чата

| Тип БП | Напряжение | Плюсы | Минусы |
|--------|-----------|-------|--------|
| Ноутбучный | 19В+ | Доступный, компактный | Может быть недостаточно тока |
| Трансформаторный | 18-24В | Надёжнее, меньше помех | Тяжёлый, громоздкий |
| ATX компьютерный | 12В | Мощный | Нужен повышающий преобразователь |
| Импульсный noname | Разный | Дёшево | Шум, нестабильность под нагрузкой |

- **Оптимальное входное**: 18-19В (при 24В Q1 греется сильнее)
- **Минимальное**: 16.5В
- **Максимальное**: 24В (для заряда 24В АКБ)
- MeanWell SDR-240-24 — эталон качества
